# Примечание о сборке установщика

## Проблема с NSIS на Linux

При сборке NSIS установщика на Linux возникает известная проблема:
- NSIS пытается включить uninstaller файл до его создания
- Это приводит к ошибке "no files found"
- Это баг в electron-builder на Linux системах
- Переопределение макросов не работает, так как electron-builder использует внутренние макросы

## Решение: Portable версия

**Portable версия работает отлично и является лучшим решением:**
- ✅ Пользователи просто запускают .exe файл (не требуется установка)
- ✅ Не требует прав администратора
- ✅ Работает стабильно на всех системах
- ✅ Удобнее для пользователей (можно запускать с флешки)
- ✅ Собирается без проблем на Linux

```bash
npm run build:portable
```

Результат: `Modern Launcher-*-portable.exe`

## Альтернативные решения для NSIS установщика

Если действительно нужен именно NSIS установщик:

1. **Сборка на Windows**: NSIS установщик собирается без проблем на Windows системе
2. **Использование Docker**: Можно использовать Windows контейнер для сборки
3. **CI/CD на Windows**: Использовать GitHub Actions или другой CI с Windows runner
4. **Виртуальная машина Windows**: Использовать VM с Windows для сборки

## Текущая конфигурация

По умолчанию `npm run build:installer` собирает portable версию, которая работает надежно на всех системах.

**Рекомендация**: Используйте portable версию - она работает отлично и даже удобнее для пользователей!
